## ___1. top 명령어___

'top'명령어는 리눅스 및 유닉스 계열 운영체제에서 실시간으로 시스템의 성능 상태와 실행 중인 프로세스를 모니터링하는 데 사용되는 유틸리티이다. 시스템 관리자와 사용자에게 CPU,메모리 사용량, 프로세스 상태 등의 정보를 실시간으로 제공하여 시스템 자원의 활용도를 파악하고, 성능 문제를 진단하는 데 도움을 준다. 


'top'명령어는 터미널에서 실행되며, 시스템의 현재 상태를 실시간으로 업데이트하여 보여준다. 상위에는 CPU 사용량과 메모리 사용량 등의 요약 정보가 나타나며, 하단에는 실행 중인 각 프로세르의 상세 정보가 표시된다. 'top'명령어에서 중요한 것은 부하가 발생하는 프로세스가 상단에 위치하므로, 상단을 주의깊게 확인할 필요가 있다.

__주요옵션__

-d : 업데이트 주기를 설정한다. 예: top -d 2 (2초마다 갱신)


-p : 특정 PID를 모니터링한다. 예: top -p 1234


-u : 특정 사용자의 프로세스만 표시한다. 예: top -u username


-n : 지정된 갱신 주기 횟수 후 종료한다. 예: top -n 10

## 요약 (top)

리눅스 운영체제에서 시스템의 성능을 모니터링하고 최적화하는 과정은 중요한 역할을 한다. 특히 모든 서비스는 자원을 이용하며 오류가 발생하거나 좀비 프로세스 발생으로 인한 자원소모율이 하드웨어의 수명을 단축시키거나 메모리, CPU 고장 등으로 인한 장비 교체를 유발할 수 있기 때문에 이 명령어는 기억해 두는게 좋다.
***

## ___2. ps 명령어___

'ps'명령어는 리눅스 및 유닉스 계열 운영체제에서 현재 실행 중인 프로세스의 상태를 보여주는 유틸리티이다. 시스템의 모든 프로세스 또는 특정 사용자나 조건에 맞는 프로세스를 확인할 수 있다. 이는 시스템 관리자와 사용자에게 유용한 정보를 제공하여 시스템 관리와 문제 해결에 도움을 준다.

__주요기능__

1. 프로세스 나열

   
   ps 명령어는 현재 실행 중인 모든 프로세스를 나열할 수 있다. 기본적으로 현재 터미널 세션에서 실행 중인 프로세스만 표시하지만, 다양한 옵션을 사용하여 더 많은 정보를 얻을 수 있다.

2. 프로세스 필터링

   
   특정 사용자나 조건에 맞는 프로세스를 필터링할 수 있다. 예를 들어, 특정 사용자가 실행 중인 프로세스만 표시하거나, 특정 프로세스 ID(PLD)에 해당하는 프로세스만 나열할 수 있다.

3. 프로세스 상태 정보 제공

   
   각 프로세스에 대한 자세한 정보를 제공한다. 여기에는 프로세스 ID(PID), 부모 프로세스 ID(PPID), 실행 중인 사용자(USER), CPU 사용률(%CPU), 메모리 사용률(%MEM), 실행 시간(TIME), 프로세스 상태(STATE) 등이 포함된다.


__주요 옵션__

-e 또는 -A : 모든 프로세스를 나열한다. 예 : ps -e

-f: 전체 포맷으로 프로세스를 나열. 자세한 정보를 제공한다. 예 : ps -f

-u [USER]: 특정 사용자의 프로세스를 나열한다. 예 : ps -u username

-p [PID]: 특정 PID에 해당하는 프로세스를 나열한다. 예 : ps -p 1234

-aux: BSD 스타일로 모든 프로세스를 나열하며, 사용자, CPU 사용률, 메모리 사용률 등을 포함한 자세한 정보를 표시한다. 예 : ps -aux

ps 명령어는 시스템 모니터링, 문제 해결, 스트립트 작성 등에 필수적인 도구로, 다양한 옵션과 출력을 조합하여 유연하게 사용할 수 있다.

## ___3. jobs 명령어___


jobs 명령어는 리눅스 및 유닉스 게열 운영체제에서 현재 쉘 세션에서 실행 중인 백그라운드 작업과 정지된 작업을 나열하는 데 사용된다. 이 명령어는 작업의 상태를 모니터링하고 관리하는 데 유용하다. 

__주요 기능__

1. 현재 쉘 세션에서  실행 중인 작업 나열
   jobs 명령어는 현재 터미널 세션에서 실행 중이거나 일시 중지된 모든 작업을 나열한다.

2. 작업 상태 확인
   각 작업의 상태를 표시한다. 상태는 Running(실행 중),Stopped(정지됨),Terminated(종료됨) 등으로 나뉜다.

__주요 옵션__


-l: 각 작업의 프로세스 그룹 ID(PGID)와 세션 ID를 포함한 상세 정보를 표시한다. 예 : jobs -l


-n: 최근에 상태가 변경된 작업만 표시한다. 예 : jobs -n


-p: 각 작업의 프로세스 ID(PID)만 표시한다. 예 : jobs -p

jobs 명령어는 현재 쉘 세션에서 실행 중인 작업을 관리하고 모니터링하는 데 매우 유용한 도구이다. 이를 통해 사용자는 백그라운드에서 실행 중인 작업과 정지된 작업의 상태를 쉽게 확인하고, 필요에 따라 작업을 재개하거나 종료할 수 있다. 다양한 옵션을 사용하여 더 많은 정보를 얻을 수 있으며, bg,fg,kill 명령어와 함께 사용하여 작업을 효과적으로 관리할 수 있다.

## ___4. kill 명령어___

kill 명령어는 리눅스 및 유닉스 운영체제에서 프로세스에 시그널을 보내기 위해 사용되는 명령어이다. 주로 프로세스를 종료하는 데 사용되지만, 다양한 시그널을 보낼 수 있어 여러 용도로 활용된다.

__주요 기능 및 시그널__

-15 (SIGTERM): 프로세스를 정상적으로 종료한다. 프로세스가 정리 작업을 수행할 수 있는 시간을 준다. 기본 시그널이다. 예 : kill -15 <PID>


-9 (SIGKILL): 프로세스를 즉시 종료한다. 정리 작업을 수행할 시간이 주어지지 않는다. 강제 종료 시 사용한다. 예 : kill -9 <PID>


-1 (SIGHUP): 프로세스를 다시 로드하거나 재시작한다. 일반적으로 설정 파일을 다시 읽도록 프로세스에 요청한다.
예 : kill -1 <PID>


-2 (SIGINT): 인터럽트 시그널을 보낸다. 일반적으로 프로세스를 중단한다. Ctrl+C를 눌렀을 때와 동일하다. 
예 : kill -2 <PID>


-3 (SIGQUIT): 종료 시그널을 보낸다. 프로세스가 코어 덤프를 생성하도록 한다. 예 : kill -3 <PID>

__시그널 이름 사용__

시그널 번호 대신 이름을 사용할 수도 있다. 시그널 이름은 대문자로 작성해야 한다.


SIGTERM 시그널 보내기 예 : kill -SIGTERM <PID>


SIGKILL 시그널 보내기 예 : kill -SIGKILL <PID>


__모든 프로세스에 시그널 보내기__

특정 사용자나 그룹의 모든 프로세스에 시그널을 보낼 수 있다.

특정 사용자의 모든 프로세스 종료 


kill -u <username>


특정 그룹의 모든 프로세스 종료


kill -g <groupname>


__프로세스 그룹에 시그널 보내기__

프로세스 그룹 ID를 사용하여 그룹에 속한 모든 프로세스에 시그널을 보낼 수 있다.

프로세스 그룹에 시그널 보내기


kill -<PGID>

__주의사항__

1. 신중한 사용

   
kill -9 명령어는 강제 종료를 의미하므로, 데이터 손실이나 시스템 불안정을 초래할 수 있다. 가능한 한 사용을 피하고, 정상 종료를 먼저 시도하는 것이 좋다


2. 루트 권한 필요

   
시스템의 주요 프로세스나 다른 사용자의 프로세스를 종료하려면 루트 권한이 필요하다.

## 요약 (kill)


kill 명령어는 시스템 관리자가 프로세스를 제어하고 시스템 성능 및 안정성을 유지하는 데 중요한 도구이다. 다양한 시그널을 통해 프로세스를 종료, 중지, 재시작할 수 있으며, pkill, killall 등의 명령어와 함께 사용하여 더욱 유연한 프로세스 관리가 가능하다
